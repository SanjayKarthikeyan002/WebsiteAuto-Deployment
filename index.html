Website Auto Deployment // AWS S3 + GitHub Actions

Project Overview

This is a static website that is automatically deployed to Amazon S3 using a CI/CD pipeline built with GitHub Actions.

Whenever I update this code and push changes to the main branch, GitHub Actions automatically uploads the latest version of this site to my S3 bucket — no manual deployment needed.

How the Deployment Works

  • Edit the website files in VS Code.
  • Commit and push the changes to GitHub.
  • GitHub Actions runs a workflow that:
    • Checks out the repository code.
    • Uses IAM credentials (stored as GitHub Secrets) to log in to AWS.
    • Runs aws s3 sync to upload files to the S3 bucket.
  • Amazon S3 is configured for static website hosting, so the new version becomes live automatically.

Tech Stack

AWS S3 (Static Website Hosting) GitHub GitHub Actions (CI/CD) IAM (Access Management) AWS CLI HTML / CSS

What I Learned

  • How to host a static website on Amazon S3.
  • How to create an IAM user with least-privilege access.
  • How to store AWS keys securely as GitHub Secrets.
  • How to build a simple CI/CD pipeline using GitHub Actions.
  • How automatic deployments reduce manual work and errors.

About This Project

This project was created to practice DevOps fundamentals without heavy coding. It focuses on automation, cloud deployment, and understanding how tools like GitHub Actions and AWS S3 work together.

Future improvements could include adding a custom domain, HTTPS using CloudFront, or deploying more complex front-end apps.
